# Project Overview
• 项目定位：一个基于ReAct（Reasoning and Acting）模式的AI驱动命令行工具（Agent），用于自动分析和理解给定的代码仓库结构、依赖、架构与核心代码逻辑。
• 核心功能：1) **智能分析Agent**：接收用户需求，遵循预设策略自动执行文件系统探索与信息收集；2) **结构化报告生成**：基于收集的信息生成包含项目模块、关键方法、技术架构等内容的分析报告。
• 技术架构：采用Go语言开发的模块化单体应用，核心依赖为`github.com/openai/openai-go`。架构上实现了ReAct模式，即Agent在AI模型的“推理”与本地工具的“执行”之间循环迭代。
• 使用场景：面向开发者或技术分析人员，旨在自动化、智能化地完成对陌生代码仓库的初步理解与文档生成工作。

# Technical Analysis
• **编程语言**：Go (Go 1.22+)
• **核心依赖**：OpenAI SDK (`github.com/openai/openai-go`) 用于与大语言模型交互。
• **数据处理与存储**：不涉及持久化存储。分析过程中的对话历史、文件内容等均保存在内存中，并通过提示词工程（上下文压缩）管理会话长度。
• **网络通信**：通过HTTP(S)与OpenAI（或兼容）API进行交互，支持流式与非流式调用。
• **关键函数入口**：`main()` -> `RunAnalysis()` -> `CodeAnalysisAgent.RunLoop()`。

# Functional Inventory
• **Agent核心引擎模块**
  • **职责**：作为系统的控制中心，实现ReAct主循环。负责管理与AI模型的对话、解析AI返回的指令、调用相应的工具执行、并维护分析上下文。
  • **核心功能点**：执行指令（`read_file`, `list_dir`, `grep`, `finish`）、管理对话历史、触发上下文压缩。
  • **依赖关系**：调用AI模型接口、文件系统工具模块。
  • **关键实现方法**：`RunLoop()`, `ExecuteCommand()`, `compressHistory()`。
  • **执行流程**：初始化 -> 发送提示词给AI -> 接收并解析命令 -> 执行工具 -> 格式化结果并构建下一轮提示 -> 循环。
  • **关键调用链**：`RunLoop` -> `aiModel.Chat` -> `ExecuteCommand` -> (`ListDir`/`ReadFileChunk`/`Grep`)。

• **AI模型交互与策略模块**
  • **职责**：集成外部大语言模型服务，并定义Agent的分析行为策略。
  • **核心功能点**：1) **模型接口**：封装对AI API的调用；2) **策略定义**：通过硬编码的提示词（`AnalysisPrompt`, `CompressionPrompt`）规定Agent的分析路径、文件读取优先级和输出格式。
  • **依赖关系**：被Agent核心引擎调用。
  • **关键实现方法**：`AIModel.Chat()`, `AIModel.ChatStream()`。核心策略常量：`AnalysisPrompt`, `CompressionPrompt`。
  • **关键调用链**：被`agent.RunLoop`调用，返回的响应被`agent`解析为命令。

• **文件系统操作工具模块**
  • **职责**：提供受控、安全的本地文件与目录操作能力，是Agent执行“行动”的物理手段。
  • **核心功能点**：列出目录内容（支持过滤）、分块读取文件内容、在文件中搜索文本模式（支持递归）。
  • **依赖关系**：被Agent核心引擎调用。
  • **关键实现方法**：`ListDir()`, `ReadFileChunk()`, `Grep()`。
  • **执行流程**：接收来自Agent的命令和参数，在限定工作目录内执行对应文件操作，返回结果或错误。

# Implementation Details
• **`main.go` (程序入口层)**
  • **职责**：解析命令行参数，初始化分析任务，启动Agent主循环。
  • **对应功能模块**：整个应用的启动器。
  • **核心入口**：`main()`函数，调用`RunAnalysis()`函数。
  • **关键实现**：定义了命令行参数（如目标目录），创建`CodeAnalysisAgent`和`OpenAI`实例，并使用`AnalysisPrompt`启动分析循环。

• **`agent.go` (核心逻辑层)**
  • **职责**：实现Agent的核心逻辑，包括ReAct循环、命令执行和上下文管理。
  • **对应功能模块**：Agent核心引擎模块。
  • **核心入口**：`RunLoop()`方法。
  • **关键实现**：
    1.  **循环控制**：持续运行直到收到`finish`命令。
    2.  **命令分发**：`ExecuteCommand()`方法根据命令名调用`file.go`中的工具函数。
    3.  **上下文管理**：`compressHistory()`方法在历史过长时调用AI模型进行总结压缩。

• **`prompt.go` (核心逻辑层)**
  • **职责**：存储定义Agent行为模式的系统提示词。
  • **对应功能模块**：AI模型交互与策略模块的策略定义部分。
  • **核心内容**：两个核心常量字符串`AnalysisPrompt`和`CompressionPrompt`，包含了完整的分析任务描述、文件读取优先级（README > 依赖文件 > 入口文件 > 核心模块）、输出格式要求以及上下文压缩指令。

• **`openai.go` (集成层)**
  • **职责**：作为与大语言模型服务通信的适配器。
  • **对应功能模块**：AI模型交互与策略模块的接口实现部分。
  • **核心入口**：`Chat()`和`ChatStream()`方法。
  • **关键实现**：实现了`AIModel`接口，使用OpenAI SDK构造请求、处理响应和流式输出。

• **`file.go` (工具层)**
  • **职责**：提供具体的文件系统操作实现。
  • **对应功能模块**：文件系统操作工具模块。
  • **核心方法**：`ListDir()`, `ReadFileChunk()`, `Grep()`。
  • **关键实现**：
    1.  **安全约束**：操作前检查目标路径是否在允许的工作目录内。
    2.  **智能过滤**：`ListDir`可过滤常见非源码文件（如图片、文档）。
    3.  **分块读取**：`ReadFileChunk`支持按行范围读取大文件。
